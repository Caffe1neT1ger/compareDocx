# Тестовые документы

Эта папка содержит тестовые DOCX документы для проверки функционала сравнения.

## Созданные документы

### test_document_1.docx
Базовый документ "Техническое задание на разработку системы" со следующей структурой:
- Раздел 1: Общие положения
  - Подраздел 1.1: Назначение системы
- Раздел 2: Требования к функциональным характеристикам
  - Глава 2.1: Управление документами
  - Глава 2.2: Поиск и фильтрация
- Раздел 3: Требования к интерфейсу
- Заключение

### test_document_2.docx
Модифицированная версия первого документа с изменениями:
- Добавлены новые абзацы и пункты
- Изменены некоторые тексты
- Добавлен новый раздел 2.3: Безопасность данных
- Расширено заключение

**Использование для тестирования:**
```bash
python main.py documents/test_document_1.docx documents/test_document_2.docx results/comparison_1_2.xlsx
```

### test_document_3.docx
Совсем другой документ "Отчет о проделанной работе" с простой структурой:
- Введение
- Раздел 1: Выполненные задачи
- Раздел 2: Результаты
- Заключение

**Использование для тестирования:**
```bash
python main.py documents/test_document_1.docx documents/test_document_3.docx results/comparison_1_3.xlsx
```

## Масштабные тестовые документы

### extended_test_document_1.docx
Большой документ "Техническое задание на разработку информационной системы управления предприятием" с расширенной структурой:
- **8 разделов** с множеством подразделов и пунктов
- **7 таблиц** с различными данными
- **6 изображений** (схемы, диаграммы)
- Полная структура технического задания

**Характеристики:**
- Содержание
- Разделы: Общие положения, Требования к системе, Функциональные требования, Технические требования, Требования к интерфейсу, Требования к безопасности, Этапы разработки, Заключение
- Множество подразделов (1.1, 1.2, 2.1, 2.2, 3.1, 3.2, и т.д.)
- Глубокие уровни вложенности (3.1.1, 3.1.2, 3.2.1, 3.2.2)

### extended_test_document_2.docx
Модифицированная версия масштабного документа с множеством изменений:
- **9 разделов** (добавлен новый раздел "Интеграция с внешними системами")
- **8 таблиц** (изменены данные, добавлены строки)
- **8 изображений** (изменены цвета, добавлены новые)
- **Множество грамматических изменений** (запятые, пунктуация)
- Изменения в текстах, значениях, структуре

**Типы изменений:**
- Грамматические: добавлены/удалены запятые, изменена пунктуация
- Содержательные: изменены значения (версии, сроки, требования)
- Структурные: добавлены новые разделы, пункты, строки в таблицах
- Визуальные: изменены изображения (цвета, текст), добавлены новые

**Использование для полноценного тестирования:**
```bash
python main.py documents/extended_test_document_1.docx documents/extended_test_document_2.docx results/comparison_extended.xlsx
```

Этот тест позволит проверить:
- Обработку большого объема данных
- Сравнение множества таблиц
- Сравнение множества изображений
- Выявление грамматических изменений
- Построение полных путей для глубокой иерархии
- Вычисление страниц для большого документа

## Генерация новых тестовых документов

### Базовые тестовые документы
Для создания базовых тестовых документов используйте скрипт:
```bash
python create_test_documents.py
```
Скрипт создаст три тестовых документа (test_document_1.docx, test_document_2.docx, test_document_3.docx).

### Масштабные тестовые документы
Для создания масштабных тестовых документов используйте скрипт:
```bash
python create_extended_test_documents.py
```
Скрипт создаст два больших тестовых документа (extended_test_document_1.docx, extended_test_document_2.docx) с множеством разделов, таблиц и изображений.
